## 検証対象 / インプット
- **検証対象 = 基本設計** (`docs/01_basic_design/*.md`)
- **目的: 機能間連携・アーキ要件が基本設計どおり繋がることを確認する**
- カバーすべき基本設計要素:
  - **システムアーキテクチャ** (`system-architecture.md`) の各 I/F (機能↔機能、機能↔外部)
  - アーキ図の **各コンポーネント境界** (API ↔ Service / Service ↔ Repository / Service ↔ 外部 SaaS 等)
  - **機能一覧** (`feature-list.md`) の **機能間依存** (F002 が F001 の出力を入力とする等)
  - **非機能要件** (`non-functional.md`) のうち結合層で検証可能なもの (応答時間 / 認可境界 / トランザクション境界 等)
- 補足: 詳細設計のシーケンス図 (`sequence.md`) は **「基本設計のどの I/F がどう動くか」の具体化** として参照する

## 方針
- 対象: 機能間連携 / 機能内の複数コンポーネント間 / 外部 I/O。
- データストアは本物 or それに近いもので実施(原則モック禁止)。
- 命名規則: `IT-{{機能ID}}-NNN`

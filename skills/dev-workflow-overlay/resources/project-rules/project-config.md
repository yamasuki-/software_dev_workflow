## ADD (ワークフロー全体への追加ルール)
- 例: PR タイトルは Conventional Commits に従う
- 例: 全関数に docstring 必須

## OVERRIDE (ベース指示の置き換え)
- 例: ベースは「機能IDは F<3桁>」だが、本プロジェクトでは「F-NNNN」(ハイフン + 4桁) に変更

## DISABLE (ベース指示の無効化)
- 例: テスト結果ドキュメントの「カバレッジ実測」セクションは不要 (別ツールで計測しているため)
